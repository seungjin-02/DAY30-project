# DAY27 — 전이 인과 고정

## 오늘 고정한 것
- 전이는 상태의 결과가 아님을 명확히 했다.
- 모든 전이는 외부 규칙 평가의 결과로만 발생하도록 인과를 고정했다.
- “~이므로”, “~하기 위해”, “사실상” 등
  전이 원인을 흐리는 언어 표현을 금지했다.

## 의도적으로 하지 않은 것
- 상태 기반 전이 설명 허용
- 인간 판단을 전이 원인으로 서술
- 전이 목적 또는 기대 효과 정의

## 설계 영향
- 전이 발생 원인이 규칙 계층으로 단일화됨
- 상태 → 판단 → 전이로 이어지는 암묵적 인과 차단
- 문서 및 README 단계에서 책임 은닉 위험 감소
