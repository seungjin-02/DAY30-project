# Transition Rules

## Scope
전이는 사전에 정의된 규칙 평가 결과로만 발생한다.

본 문서에 정의되지 않은 사유로는 어떠한 상태 전이도 발생하지 않는다.

---

## Transition Cause

전이는 다음 규칙 범주에 대한 평가 결과일 때만 발생한다.

- Threshold-based Rules  
  - 관측 가능한 수치 값이 사전에 고정된 임계값을 초과하거나 미만인 경우
  - 임계값은 런타임에서 변경되지 않는다

- Time-based Rules  
  - 특정 상태가 정의된 시간 이상 유지된 경우
  - 시간 경과는 조건 입력값으로만 사용된다

- Consistency / Integrity Rules  
  - 상태 간 논리적 불일치가 발생한 경우
  - 필수 상태 값 누락 또는 무결성 위반이 발생한 경우

- External Validity Rules  
  - 외부 입력이 사전 정의된 스키마를 벗어난 경우
  - 외부 이벤트의 출처 또는 형식이 검증되지 않은 경우

---

## Prohibited Causes

다음 항목은 상태 전이의 원인이 아니다.

- 상태의 의미적 해석
- 상태 변화의 목적적 해석
- 모델 기반 추론 결과
- 인간 판단이 명시적 규칙으로 정의되지 않은 모든 개입

---

## Enforcement Rules

- 하나의 입력이 복수 규칙 범주에 동시에 해당할 경우 자동 전이는 중단된다.
- 규칙 평가 결과가 불확실하거나 판정 불가한 경우 자동 전이는 수행되지 않는다.
- 중단 이후 전이는 HUMAN_REQUIRED 상태로만 진행된다.
